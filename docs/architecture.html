The Learning Loop

This is the core insight. GaryClaw isn't a pipeline — it's a feedback loop. Outcomes feed back into decisions. Failures inform what to build next. The daemon literally improves itself.

Pipeline Composer selects minimal skill sequence per task (XS → $0.50, M → $4). Oracle learns from per-category outcomes. When backlog empties, the invention critique protocol activates.

Deep Prioritize (Session 4 — human-built)

Failure Pattern Analysis
Reads failures.jsonl across all instances. Counts by category + skill. +2 bonus for items fixing recurring failures.
Decision Quality Trends
Oracle metrics + low-confidence topic clustering via groupDecisionsByTopic(). +1 bonus for domain knowledge.
Impact Measurement
Compares avg cost recent vs older jobs. Cost up → optimize. Cost down → build features.

Invention Critique Protocol

1. Research gaps
→
2. Invent 3-5
→
3. Critique each
→
4. Prune <7
→
5. Score & pick

Pride test: "Would I be proud to show this to the project owner?" Below 7 → discard. None survive → think bigger.

Oracle Skill Selection (Session 4 — daemon-built)

Mode 1: Deterministic
--todo "Title" skill1 skill2
Human owns the pipeline. Bypasses composition entirely.
Mode 2: Oracle Analysis
Task category → per-category outcomes → skill catalog → Oracle recommends. Learning loop.
Fallback: Static Table
(effort, priority, hasDesignDoc) → minimal skills. Unknown skills pass through.
