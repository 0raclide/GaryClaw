## Problem Statement

4 remaining P5 QA findings from prior runs. 11 of 15 sub-items already fixed. These are the last cleanup items preventing the P5 tier from being fully retired:

1. `isTaste` field in `OracleOutput` mirrors confidence threshold, not taste semantics
2. `parseDomainSections` missing test for adjacent sections without body text
3. Pipeline `startTime!` non-null assertion at `pipeline.ts:413`
4. Pipeline static import of orchestrator creates tight coupling between top-level modules

## What Makes This Cool

Backlog hygiene. Retiring the entire P5 tier means every future prioritize run has a cleaner signal-to-noise ratio. Four XS fixes, each independently testable, each making the codebase more honest about what it does.

## Constraints

- Each fix must be backward-compatible (no interface changes that break existing callers)
- Tests must cover the exact edge case, not just the happy path
- No functional behavior changes, only correctness and clarity improvements

## Premises

1. `isTaste` should be renamed to `isLowConfidence` since it is set to `parsed.confidence < config.escalateThreshold` (oracle.ts:139) and has nothing to do with taste detection. Verified: 10 source references, 18 test references across 6 files.
2. The `parseDomainSections` adjacent-sections-without-body edge case IS already handled by the `isBodyActuallyFrontmatter` heuristic (researcher.ts:289), but has no dedicated test. The logic is correct. We just need to prove it.
3. `entry.startTime!` at pipeline.ts:413 is always set at line 305, but the type says `startTime?: string`. A `??` default eliminates the assertion without changing behavior.
4. Pipeline imports orchestrator both statically (line 16: `import { runSkill }`) and dynamically (line 466: `await import("./orchestrator.js")`). Both can be dynamic since a dynamic import already exists in this file. DI is overkill for a single-repo project.

## Approaches Considered

### Approach A: Direct Fixes (Minimal Viable)

Fix each item in-place with the smallest possible diff.

**Summary:** Rename `isTaste` to `isLowConfidence` across oracle.ts + ask-handler.ts + all tests. Add one test for adjacent sections. Replace `startTime!` with `entry.startTime ?? new Date().toISOString()`. Convert static orchestrator import to dynamic.

**Effort:** XS (4 independent changes, ~30 lines of source, ~40 lines of test)
**Risk:** Low
**Pros:**
- Smallest diff, fastest to land
- Each fix is independently reviewable
- Zero behavior change

**Cons:**
- Renaming `isTaste` touches 10 source locations + 18 test references (mechanical but wide)
- Both orchestrator imports become dynamic (lazy load on first call)

**Reuses:** Existing test patterns in researcher.test.ts, oracle.test.ts, pipeline.test.ts

### Approach B: Dependency Injection for Fix 4 (Ideal Architecture)

**Summary:** Same as Approach A for fixes 1-3, but Fix 4 extracts `RunSkillFn`/`RunSkillWithPromptFn` types into types.ts, adds parameters to `runPipeline`/`resumePipeline`.

**Effort:** S (adds ~25 lines for types + signature changes + caller updates)
**Risk:** Low-Med
**Pros:**
- Makes the dependency explicit and testable (can mock `runSkill` in pipeline tests)
- Architecturally cleaner long-term

**Cons:**
- Changes `runPipeline` and `resumePipeline` signatures (touches cli.ts, job-runner.ts)
- Over-engineered for a single-repo project with no external consumers
- More test changes needed to pass the new parameters

## Recommended Approach

**Approach A: Direct Fixes.** GaryClaw has no external consumers. The `isTaste` rename is mechanical, the test is additive, the `startTime` fix is a one-liner, and converting to dynamic import is consistent with the pattern already in this file. No DI needed.

### Fix 2: Add `parseDomainSections` Adjacent Sections Test

**File:** `test/researcher.test.ts` (add to existing `parseDomainSections` describe block)

```typescript
it("handles adjacent sections without body text", () => {
  // Two frontmatter blocks back-to-back, no body between them
  const content = [
    "---",
    "topic: WebSockets",
    "last_researched: 2026-03-25T00:00:00Z",
    "search_count: 2",
    "---",
    "topic: OAuth 2.1",
    "last_researched: 2026-03-26T00:00:00Z",
    "search_count: 1",
    "---",
    "## OAuth body here",
  ].join("\n");

  const sections = parseDomainSections(content);
  expect(sections).toHaveLength(2);
  expect(sections[0].topic).toBe("WebSockets");
  expect(sections[0].content).toBe("");  // No body for first section
  expect(sections[1].topic).toBe("OAuth 2.1");
  expect(sections[1].content).toContain("OAuth body here");
});
```

**Verification:** `npm test -- test/researcher.test.ts`

### Fix 3: Replace `startTime!` with Explicit Default
