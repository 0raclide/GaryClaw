## Problem Statement

`buildOraclePrompt()` (line 137) and `buildBatchOraclePrompt()` (line 369) in `src/oracle.ts` share ~40 lines of identical prompt-building logic. The shared prefix assembles: system preamble, Decision Principles injection, Current Context (skill name, project), Oracle Memory injection (taste, domain expertise, decision outcomes, MEMORY.md), and Recent Decisions.

The comment on line 381 literally says `// Inject oracle memory (same as single prompt)`, acknowledging the copy-paste. Any change to memory injection, principles formatting, or context assembly requires updating both functions in sync. In a system that autonomously modifies its own Oracle prompts, this is a guaranteed divergence bug.

## What Makes This Cool

It's the smallest possible change that de-risks the next high-value item (Oracle Session Reuse, P3). That feature needs a clean prompt prefix to build on. Without this fix, Oracle Session Reuse would need to patch two functions, which doubles the bug surface for a latency optimization that touches every Oracle call.

Also: the daemon built the batch Oracle prompt by copy-pasting the single prompt. Now the daemon gets to clean up its own mess. Self-improving system cleaning up its own technical debt. That's the whole point.

## Constraints

- Zero behavior change. Every test must pass without modification (pure refactor).
- No new files. The helper lives in `src/oracle.ts` alongside its callers. The new `OraclePromptPrefixInput` interface stays in `oracle.ts` (not `types.ts`) because it is internal to the oracle module and not referenced by other modules. If Oracle Session Reuse later needs it externally, move to `types.ts` at that point.
- No new dependencies. No new types unless they simplify the interface.
- Existing test assertions on prompt content must remain valid (they check substring presence).

## Premises

1. The ~40-line prefix is genuinely identical between both functions and will remain so. Verified by diff.
2. A single extracted function is the right abstraction level. Two callers does not justify a class, builder pattern, or template system. (P5: Explicit over clever.)
3. Both `OracleInput` and `OracleBatchInput` share the fields needed for the prefix (`skillName`, `projectContext`, `memory`, `decisionHistory`), so a common input type for the prefix is natural.
4. Existing tests pass unchanged because they test the public functions (`buildOraclePrompt`, `buildBatchOraclePrompt`) which still produce the same output, just assembled differently internally.
5. Doing nothing has real cost: Oracle Session Reuse (next P3 item) will need to modify the prompt prefix in two places.

## Approaches Considered

### Approach A: Extract buildOraclePromptPrefix()

Pull the shared ~40 lines into a new exported function `buildOraclePromptPrefix()` that returns the prefix string. Both `buildOraclePrompt` and `buildBatchOraclePrompt` call it, then append their own question/instructions section.

**New function signature:**
```typescript
interface OraclePromptPrefixInput {
  skillName: string;
  projectContext?: string;
  memory?: OracleMemoryFiles;
  decisionHistory: Decision[];
}

export function buildOraclePromptPrefix(input: OraclePromptPrefixInput): string
```

Both callers pass their existing fields through. The prefix returns everything up to and including Recent Decisions. Each caller then appends its own question format and response instructions.

- Effort: XS (5 min CC time)
- Risk: Low
- Pros: Single source of truth, zero behavior change, independently testable, Oracle Session Reuse can hook directly into prefix
- Cons: One more export (trivial)

### Approach B: Shared PromptBuilder class

Define a `PromptBuilder` class with methods like `.addPreamble()`, `.addMemory()`, `.addDecisions()`, `.addQuestion()`. Both functions instantiate the builder and call the relevant methods.

- Effort: S (15 min CC time)
- Risk: Low
- Pros: More extensible for hypothetical future Oracle variants
- Cons: Over-abstraction for two callers, YAGNI, more test surface, violates P5 (explicit over clever)

## Recommended Approach

**Approach A: Extract buildOraclePromptPrefix()**

Rationale: It's the minimal change that eliminates the duplication. Two callers does not justify a class hierarchy. P4 (DRY) is satisfied by the extraction. P5 (Explicit over clever) says a plain function beats an abstraction. The new function is a clean hook point for Oracle Session Reuse.

### Step 2: Extract buildOraclePromptPrefix()

Move the shared logic (system preamble, principles, context, memory injection, recent decisions) into a new exported function:

```typescript
export function buildOraclePromptPrefix(input: OraclePromptPrefixInput): string {
  let prompt = `You are a decision-making oracle for GaryClaw, an autonomous development tool.

## Decision Principles
${DECISION_PRINCIPLES}

## Current Context
- Skill: /${input.skillName}
${input.projectContext ? `- Project: ${input.projectContext.slice(0, 500)}` : ""}

`;

  if (input.memory) {
    const { taste, domainExpertise, decisionOutcomes, memoryMd } = input.memory;
    if (taste) {
      prompt += `## Taste Profile (personal preferences)\n${taste}\n\n`;
    }
    if (domainExpertise) {
      prompt += `## Domain Expertise (researched knowledge)\n${domainExpertise}\n\n`;
    }
    if (decisionOutcomes) {
      prompt += `## Decision Outcomes (what worked and what didn't — P7 applies here)\n${decisionOutcomes}\n\n`;
    }
    if (memoryMd) {
      prompt += `## Project Memory (MEMORY.md)\n${memoryMd}\n\n`;
    }
  }

  if (input.decisionHistory.length > 0) {
    const recent = input.decisionHistory.slice(-5);
    prompt += `## Recent Decisions (last ${recent.length})\n`;
    for (const d of recent) {
      prompt += `- Q: "${d.question}" → A: "${d.chosen}" [${d.principle}]\n`;
    }
    prompt += "\n";
  }

  return prompt;
}
```

### Step 3: Refactor buildOraclePrompt() to use prefix

```typescript
export function buildOraclePrompt(input: OracleInput): string {
  let prompt = buildOraclePromptPrefix(input);

  const hasOtherOption = input.options.some(
    (o) => o.label.toLowerCase() === "other",
  );

  prompt += `## Question
${input.question}

## Options
${input.options.map((o, i) => `${i + 1}. **${o.label}**: ${o.description}`).join("\n")}

## Instructions
Choose the best option using the Decision Principles above. Consider consistency with prior decisions.
${input.memory?.taste ? "Also consider the Taste Profile preferences above when they are relevant." : ""}

Respond with ONLY a JSON object (no markdown fences, no explanation outside the JSON):
{
  "choice": "<exact label of chosen option>",
  "confidence": <1-10>,
  "rationale": "<one sentence explaining why>",
  "principle": "<which of the 7 principles drove this decision>"${hasOtherOption ? `,
  "otherProposal": "<if choice is 'Other', provide a detailed free-text proposal here>"` : ""}
}`;

  return prompt;
}
```

### Step 4: Refactor buildBatchOraclePrompt() to use prefix

```typescript
export function buildBatchOraclePrompt(input: OracleBatchInput): string {
  let prompt = buildOraclePromptPrefix(input);

  // List all questions (batch-specific from here down)
  prompt += `## Questions (answer ALL ${input.questions.length} questions)\n\n`;

  for (const q of input.questions) {
    const hasOtherOption = q.options.some(
      (o) => o.label.toLowerCase() === "other",
    );

    prompt += `### Question ${q.id}\n${q.question}\n\nOptions:\n`;
    prompt += q.options.map((o, i) => `${i + 1}. **${o.label}**: ${o.description}`).join("\n");
    prompt += "\n";
    if (hasOtherOption) {
      prompt += `(If choosing "Other" for Q${q.id}, include an "otherProposal" field)\n`;
    }
    prompt += "\n";
  }

  const hasTaste = input.memory?.taste;

  prompt += `## Instructions
Answer ALL ${input.questions.length} questions using the Decision Principles above. Consider consistency across answers.
${hasTaste ? "Also consider the Taste Profile preferences above when they are relevant." : ""}

Respond with ONLY a JSON array (no markdown fences, no explanation outside the JSON).
Each element corresponds to one question, in order:
[
  {
    "questionId": 1,
    "choice": "<exact label of chosen option>",
    "confidence": <1-10>,
    "rationale": "<one sentence explaining why>",
    "principle": "<which of the 7 principles drove this decision>"
  },
  ...
]`;

  return prompt;
}
```
