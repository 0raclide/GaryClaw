## Problem Statement

The Live Evolution Dashboard is 95% built. Backend (dashboard-server.ts, 1099 lines), frontend (all 4 views in app.js, 739 lines), CLI wiring (`dashboard --serve --port --open`), SSE streaming, growth pipeline, mutation timeline, activity feed, taste/expertise rendering... all implemented. QA fixed 3 issues. 1548 lines of tests across 3 files.

But a smoke test reveals a critical data quality bug: the growth chart and module attribution show **zero daemon commits** and **zero daemon-built modules**. The CLAUDE.md claims "96% of commits are daemon-generated" and "The daemon built 22 of them itself." The actual git history has 329/798 commits (41%) with `garyclaw-daemon@local` as the *committer* email. The bug: `buildGrowthSnapshots()` and `buildModuleAttribution()` check `%ae` (author email) but the daemon sets `GIT_COMMITTER_EMAIL`, not `GIT_AUTHOR_EMAIL`. Every daemon commit looks human-authored.

This is the hero chart. The growth view with daemon vs human bars is the screenshot that gets shared. Showing 0% daemon when the real number is 41% defeats the whole purpose.

Secondary: CLAUDE.md doesn't list the dashboard as COMPLETE or include it in the module map. The TODO item should be closed.

## What Makes This Cool

Fix the attribution, and the growth chart comes alive. You see the first 200 commits are all human (blue). Then the daemon starts contributing. By the most recent day, 41% of bars are green. The module attribution grid goes from "all human" to "22 daemon-built." This is the visual story of a system learning to build itself.

## Constraints

- No new modules. Fix in existing `dashboard-server.ts` (~5-line change)
- No new external dependencies
- Must not break the 3 existing test files (1548 lines)
- Growth cache must invalidate after the fix (different data shape)
- CLAUDE.md update is non-breaking metadata

## Premises

1. **The bug is a 5-line fix.** Change `%ae` to `%ce` (committer email) in two git log format strings inside `buildGrowthSnapshots()` and `buildModuleAttribution()`. The daemon sets `GIT_COMMITTER_EMAIL = "garyclaw-daemon@local"` via `buildSdkEnv()` in sdk-wrapper.ts. Committer email is the correct signal for daemon attribution.

2. **The growth cache must be invalidated.** The cached `growth-cache.json` has the wrong data (0 daemon commits). After the fix, the cache key (HEAD SHA) won't change, so the stale cache would persist. Either add a version field to the cache schema or delete the cache file in the fix commit.

3. **The test suite needs updating.** `dashboard-server-growth.test.ts` (388 lines) mocks git output. The mock format strings use `%aI %ae` and must change to `%aI %ce` (or whichever the fix uses). Module attribution tests similarly need format string updates.

4. **CLAUDE.md update is the "mark complete" step.** Add "Live Evolution Dashboard: COMPLETE" to Current Status, add `dashboard-server.ts` full description to Module Map, update meta-story stats if needed.

5. **No frontend changes needed.** The frontend already renders daemon vs human correctly (green badge for "daemon", blue badge for "human" in module attribution; bar chart tooltips show "N human, M daemon"). The data just needs to be correct.

## Recommended Approach

**Approach B: Fix Attribution + Hybrid Module Counting.** The attribution fix is mandatory (the chart is factually wrong). The hybrid module counting is the difference between a chart that jumps from 8 to 43 in one day and one that shows gradual growth. The growth chart is the hero visual. It should tell the real story.

The approach:
1. In `buildGrowthSnapshots()`: change `--format=%H %aI %ae` to `--format=%H %aI %ce` (line ~894). Change the cumulative commit counter to use `%ce` too (line ~904).
2. In `buildModuleAttribution()`: change `--format=%ae` to `--format=%ce` (line ~975).
3. Add conditional `git ls-tree --name-only {sha} src/` count as fallback: run only for snapshots where `extractCounts()` returned null (CLAUDE.md didn't contain the "N source modules" pattern at that commit). Do not use a hardcoded threshold. Cache ls-tree results in the `GrowthCache` structure to avoid re-computing on subsequent loads.
4. Add a `cacheVersion` field to `GrowthCache` interface. Set to `2`. Old caches without version or version < 2 get rebuilt.
5. Update tests in `dashboard-server-growth.test.ts` to use `%ce` format strings.
6. Update CLAUDE.md: add "Live Evolution Dashboard: COMPLETE" to Current Status, update module map description for `dashboard-server.ts`.
